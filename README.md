# Step 3

## Project Structure
`IFID` contains the (Fetch + Decode) testbench.
`EX` contains the Execute Stage testbench.
`MEMWB` contains the (Memory + Write Back) testbench.
`CU` contains the Control Unit testbench.

Inside each one, there are 3 folders:
- `sim`: Contains scripts for automated compilation & simulation. The testbench can be compiled by running `./run.sh`.
- `src`: Contains source (VHDL) designs to verify (DUT).
- `tb`: Contains SystemVerilog UVM testbench files for DUT.
- `syn`: Not tracked and initially not present, gets created when a design is synthesized.

## Folders and code notation
The following rules were adopted while developing the project, and especially the code, to make it more readable, hopefully!

### Files:
- Files inside `sim/`, `src/` and `syn/` do not have any specific notation
- **SV Modules**: `lowerCamelCase`
- **SV Classes/Interfaces** : `UpperCamelCase` (same as class name)

### Code:
- **Constants**: `UPPER_SNAKE_CASE`
- **UVM names, methods, classes**: `lower_snake_case`
- **Custom derived classes names and interfaces**: `UpperCamelCase` (same as file name)
- **Custom derived classes methods and variables**: `lowerCamelCase`
- **Processes**: `PROC_UpperCamelCase`

## Compilation and simulation
The main script which handles every step is `sim/run.sh`.

### Utility Scripts
Inside `sim/`, there are also other custom scripts i developed for portability across different environments, allowing whoever to have the same output (even colors!) as the ones i used during development:
- `playground_run.sh`: Modified version of `run.sh` which compiles and simulates the testbench on [EDA Playground](https://edaplayground.com). Could be useful if the server is temporarily down.
- `vsim_run_first_sim.do`: TCL script only used when running QuestaSim's GUI to add waves to the waveform and color them.
- `sim_colors.sh`: Simple bash script which defines some `echo` wrappers for printing colored text on the terminal. It also defines a `colorize` function which is able to color every EDA tool's output to the user's liking.
- `systemverilog_utils.sh`: Contains just one bash function which extracts from the Top Level Testbench file the name of the top level module. It gets later used in `vopt`/`vsim` commands.

### Main run scripts
Two scripts have been developed, one for running an RTL simulation of the DUT and reporting the corresponding coverage (`run_rtl.sh`), and the other for synthesizing the DUT and then performing post-synthesis simulation of the Gate-level netlist, also reporting coverage (`run_syn.sh`).

Both scripts are divided by comments in different steps:
1. **Script includes and directory**: Here i defined most of variables used after in the script. There are also 2 functions:
  - `compile_files`: Compiles every kind of filetype (VHDL, Verilog, SystemVerilog), also allowing mixed-language simulation later on.
  - `cov_print`: Takes coverage percentage info from reports (generated by `coverage report` later on in the script) and prints it with colors on screen.

2. **Source and Testbench Files Compilation**: Here, i first compile VHDL and Verilog source files get compiled into the general `work` library, and then SystemVerilog testbench files, into the same library.

3. **Coverage Types**: I then define some coverage commands to build an all-encompassing command to later use in `vsim`. These allow me to create various files inside `sim/coverage/` directory, each of which contains its specific coverage type. A "global" coverage file is also created.

4. **Design optimization with Coverage tracking**: Here i use `vopt` command to optimize the testbench design and track every kind of coverage type.

5. **Simulation**: Here i define some parameters (`SIM_...`, which can be modified) later used in `vsim`, and i the simulate using the latter command.

6. **Coverage reporting**: This last section basically takes every coverage file created before, and outputs everything on screen. Looking at the raw report files is left for "debugging" low coverage. A coverage report for each type of coverage gets created inside the `sim/coverage` directory. They are mainly used for "debugging" coverage and improving it, while the "cov_all.txt" report contents get extracted from the script and then output to the screen.
